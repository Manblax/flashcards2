# Документация компонентов

## Структура компонентов

### Layout компоненты

#### MainLayout
Основной layout приложения, который объединяет Sidebar и Header.

**Расположение:** `components/layout/MainLayout.tsx`

**Использование:**
```tsx
import MainLayout from "@/components/layout/MainLayout";

<MainLayout>
  {children}
</MainLayout>
```

#### Sidebar
Боковая панель навигации с логотипом и меню.

**Расположение:** `components/layout/Sidebar.tsx`

**Особенности:**
- Использует `usePathname` для подсветки активного пункта меню
- Адаптивная навигация
- Иконки для каждого пункта меню

#### Header
Шапка приложения с поиском, кнопкой создания модуля и аватаром пользователя.

**Расположение:** `components/layout/Header.tsx`

**Особенности:**
- Sticky позиционирование
- Кнопка создания модуля (круглая с иконкой плюса)
- Поле поиска
- Аватар пользователя

### Компоненты модулей

#### ModuleCard
Карточка модуля для отображения в списках.

**Расположение:** `components/ModuleCard.tsx`

**Props:**
```typescript
interface ModuleCardProps {
  module: Module;
}
```

**Особенности:**
- Горизонтальный layout с иконкой слева
- Отображает название, количество терминов и автора
- Правильное склонение слова "термин"
- Hover эффекты
- Ссылка на страницу модуля

#### InfiniteModuleList
Компонент списка модулей с бесконечной прокруткой.

**Расположение:** `components/InfiniteModuleList.tsx`

**Props:**
```typescript
interface InfiniteModuleListProps {
  allModules: Module[];
}
```

**Особенности:**
- Загружает 20 модулей за раз
- Использует Intersection Observer API для отслеживания прокрутки
- Группирует модули по временным периодам (эта неделя, прошлая неделя, по месяцам)
- Индикатор загрузки
- Сообщение о завершении списка

## Типы данных

### Module
**Расположение:** `types/module.ts`

```typescript
export interface Module {
  id: string;
  title: string;
  termCount: number;
  author: string;
  createdAt: Date;
  updatedAt: Date;
}
```

## Утилиты

### mockData
**Расположение:** `lib/mockData.ts`

Содержит функции для генерации моковых данных:
- `generateMockModules(count: number)` - генерирует указанное количество модулей
- `mockModules` - экспортированный массив из 100 модулей

## Страницы

### Главная (/)
**Файл:** `app/page.tsx`

Отображает 6 последних модулей в сетке 2 колонки.

### Библиотека (/library)
**Файл:** `app/library/page.tsx`

Страница с табами (сейчас только "Модули").
Отображает все модули с бесконечной прокруткой, сгруппированные по периодам.

### Создание модуля (/create)
**Файл:** `app/create/page.tsx`

Форма создания нового модуля (заглушка).

### Просмотр модуля (/module/[id])
**Файл:** `app/module/[id]/page.tsx`

Страница просмотра конкретного модуля (заглушка).

## Стилизация

### Цветовая схема (DaisyUI темная тема)
- **Primary:** `#6366f1` (индиго)
- **Base-100:** `#0a0e27` (темно-синий фон)
- **Base-200:** `#141937` (немного светлее для sidebar)
- **Base-300:** `#1e2549` (для карточек)
- **Neutral:** `#2a3256` (для границ и вторичных элементов)
- **Neutral-content:** `#a6adbb` (для вторичного текста)

### Основные утилиты Tailwind
- `rounded-2xl` - для карточек
- `rounded-xl` - для иконок
- `rounded-lg` - для кнопок
- `gap-4`, `gap-6` - для сеток
- `hover:` префикс для интерактивных элементов

## Иконки

Используются SVG иконки, встроенные в компоненты:
- Иконка модуля (сетка)
- Иконка поиска
- Иконка плюса (создание)
- Иконка меню (гамбургер)
